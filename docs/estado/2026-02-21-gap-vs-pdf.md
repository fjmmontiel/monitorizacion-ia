## Alineado con el plan
- Arquitectura 1 FE + 1 BE orquestador.
- Endpoints canónicos activos:
  - `POST /cards?caso_de_uso=...`
  - `POST /dashboard?caso_de_uso=...`
  - `POST /dashboard_detail?caso_de_uso=...&id=...`
  - `GET /health`
- Front monitor con composición `cards + tabla + detalle`.
- Contratos tipados y validación de respuestas.
- Configuración por caso de uso y adapters en backend.

## Diferencias deliberadas
- Parametrización final de contrato en código: `caso_de_uso` (el PDF usa `system_name`).
- Scope actual centrado en fase ejecutable local + perfiles duales (local/corporativo), sin hardening de seguridad corporativa reintroducida en runtime.
- E2E automatizado implementado como smoke + casos multi-configuración por scripts shell (no suite Cypress corporativa completa en esta fase).

## Extensiones añadidas para operativa local
- `home /monitor` con sistemas navegables como tabs (desde configuración de casos de uso).
- Concepto de `vista` configurable por JSON:
  - `operativa`
  - `supervision`
  - `ejecutiva`
- Casos E2E multi-configuración (`caso_de_uso + vista + query params`) listos para ejecutar.

## Pendientes conocidos
- Toolchain de tests FE corporativos no ejecutable en este equipo por preset `@architecture-components/test-config` ausente en instalación shell actual.
- `lerna` no disponible en `.bin` de raíz, por lo que validaciones se ejecutan por package (`packages/shell`) y scripts locales.
